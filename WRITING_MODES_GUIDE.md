# Writing Modes Guide

A comprehensive guide to all 11 Writing Modes available in the Django Writing Platform. Each mode provides specialized tools and techniques to enhance different aspects of the writing process.

---

## üìù Overview

The Writing Modes system offers writers a comprehensive toolkit for analyzing, editing, and improving their documents. Each mode focuses on specific aspects of writing, from basic editing to advanced rhetorical analysis.

### Mode Categories

- **Structure & Organization**: Textris, Fire, Meta
- **Reading & Analysis**: Reader, Infrared, Word Cloud
- **Academic Writing**: Source Synthesis, Key Terms
- **Audience & Perspective**: Audience Mode
- **Editing & Revision**: Greening Mode
- **Productivity**: Pomodoro Mode

## 1. üß© Textris Mode
*Interactive sentence and paragraph manipulation*

### Purpose
Enable writers to easily reorganize their document structure by moving sentences and paragraphs around like Tetris blocks.

### Key Features
- **Two manipulation levels**: Sentence-level and paragraph-level editing
- **Visual block interface**: Text segments appear as moveable blocks
- **Drag-and-drop functionality**: Intuitive reordering mechanism
- **Undo/Redo system**: Full history tracking for safe experimentation
- **Live preview**: Real-time document updates as you move blocks

### How It Works
1. Activate Textris Mode to transform text into moveable blocks
2. Choose between sentence or paragraph manipulation level
3. Drag and drop blocks to reorganize content
4. Use undo/redo buttons to experiment safely
5. Exit mode to finalize changes

### Best Used For
- Restructuring arguments and ideas
- Finding better logical flow
- Experimenting with document organization
- Breaking writer's block through physical manipulation

## 2. üî• Fire Mode
*Identify and position key sentences for maximum impact*

### Purpose
Help writers identify their most important sentences and strategically position them for maximum rhetorical impact, based on writing pedagogy principles.

### Key Features
- **Two-stage process**: Selection phase followed by restructuring phase
- **Key sentence identification**: Highlight the most important sentences
- **Strategic positioning**: Move key sentences to impactful positions
- **Visual feedback**: Selected sentences remain highlighted throughout
- **Paragraph restructuring**: Reorganize around key sentences

### How It Works
**Stage 1 - Selection:**
1. Read through document and click on key sentences
2. Build a collection of the most important ideas
3. View selection counter and manage chosen sentences

**Stage 2 - Restructuring:**
1. Move to restructuring phase with selected sentences highlighted
2. Reorganize paragraphs around key sentences
3. Position important ideas at paragraph beginnings or ends
4. Finalize the improved structure

### Best Used For
- Academic papers requiring strong thesis statements
- Persuasive writing with key arguments
- Improving paragraph topic sentences
- Strengthening document flow and impact

## 5. üéØ Key Terms Mode
*Custom term highlighting with color coding*

### Purpose
Enable writers to define and visually track important terms, concepts, or themes throughout their document using customizable highlighting.

### Key Features
- **Custom term groups**: Define multiple sets of related terms
- **Color-coded highlighting**: Assign unique colors to each term group
- **Wildcard support**: Use `*` for pattern matching (e.g., `analy*` matches "analyze", "analysis")
- **Bulk term management**: Add multiple terms separated by commas
- **Persistent highlighting**: Terms remain highlighted across editing sessions

### How It Works
1. Activate Key Terms Mode
2. Click "Edit Terms" to open the terms panel
3. Add term groups with associated colors
4. Enter terms separated by commas (wildcards supported)
5. Apply highlighting to see terms throughout document
6. Use "Clear Highlighting" to remove all highlights

### Best Used For
- Tracking key concepts in academic papers
- Ensuring consistent terminology usage
- Analyzing theme distribution throughout text
- Identifying overused or underused terms

## 6. üå°Ô∏è Infrared Mode
*Reveal hidden language patterns and writing issues*

### Purpose
Make visible the "invisible" aspects of writing by highlighting specific language patterns that affect clarity and style.

### Key Features
- **Pattern detection**: Multiple pre-configured language patterns
- **Jargon identification**: Highlights complex Latinate vocabulary
- **Nuance markers**: Identifies words that signal subtlety or qualification
- **Transition words**: Shows logical connectors and flow markers
- **Custom patterns**: Ability to add custom word lists
- **Pattern statistics**: Count and analyze pattern distribution

### Available Patterns
- **Jargon Words (Latinate)**: Complex, academic vocabulary that might confuse readers
- **Nuance Words**: Terms that indicate subtlety, qualification, or complexity
- **Transition Words**: Connectors that show logical relationships
- **Hedge Words**: Terms that soften or qualify statements

### How It Works
1. Activate Infrared Mode
2. Select pattern type from dropdown menu
3. View highlighted instances throughout document
4. Use pattern statistics to analyze distribution
5. Export analysis for further review

### Best Used For
- Simplifying academic writing for broader audiences
- Identifying overuse of hedge words or qualifiers
- Checking transition word usage for logical flow
- Making technical writing more accessible

## 8. üå± Greening Mode
*John McPhee's text reduction technique*

### Purpose
Implement the "greening" editing technique popularized by John McPhee, helping writers reduce text length while preserving voice, meaning, and tone.

### Key Features
- **Goal setting**: Define reduction targets by percentage, word count, or line count
- **Text selection**: Double-click or drag to select text for removal
- **Visual feedback**: Selected text appears with green highlighting
- **Progress tracking**: Real-time progress toward reduction goal
- **Reversible selection**: Click highlighted text to deselect
- **Safe application**: Confirmation dialog before permanent deletion

### Goal Types
- **Percentage reduction**: Reduce document by 10-50%
- **Word count**: Remove specific number of words
- **Line count**: McPhee's traditional line-based approach

### How It Works
1. Activate Greening Mode
2. Set reduction goal in the modal dialog
3. Select text for deletion by double-clicking or dragging
4. Monitor progress with real-time feedback
5. Use "Clear Selections" to start over if needed
6. Apply greening when satisfied with selections

### Best Used For
- Meeting strict word limits for publications
- Improving conciseness and clarity
- Removing redundancy and wordiness
- Learning disciplined editing techniques

## 9. üîó Source Synthesis Mode
*Track source interactions in academic writing*

### Purpose
Help academic writers analyze how their sources interact throughout their document, identifying whether sources are used in isolation or in rich dialogue with each other.

### Key Features
- **Source definition**: Create named sources with associated terms/concepts
- **Color-coded highlighting**: Each source gets a unique color
- **Integration analysis**: Identify single-source vs. multi-source paragraphs
- **Statistical reporting**: Generate detailed integration analysis
- **Export functionality**: Download comprehensive analysis reports
- **Wildcard support**: Use `*` for flexible term matching

### Analysis Metrics
- **Single-source paragraphs**: Sections dominated by one source
- **Multi-source paragraphs**: Areas where sources are in dialogue
- **No-source paragraphs**: Sections lacking source attribution
- **Integration ratio**: Percentage of paragraphs with multiple sources

### How It Works
1. Activate Source Synthesis Mode
2. Define sources with names and associated terms
3. Assign colors to each source for visual tracking
4. Apply highlighting to see source distribution
5. Review integration statistics
6. Export analysis report for further review

### Best Used For
- Literature reviews requiring source synthesis
- Research papers with multiple competing theories
- Ensuring balanced source usage across document
- Improving academic argument development

## üîß Technical Implementation

### Architecture
Each Writing Mode is implemented as:
- **Individual JavaScript class** in `/static/js/modes/`
- **Dedicated CSS stylesheet** in `/static/css/modes/`
- **Centralized management** through `WritingModesManager`
- **Django template integration** for seamless loading

### Key Features
- **Modular design**: Each mode is self-contained and independent
- **Consistent UI**: Unified interface patterns across all modes
- **Data persistence**: Settings and annotations saved per document
- **Export functionality**: Most modes support data export
- **Responsive design**: Mobile-friendly interfaces
- **Accessibility**: ARIA labels and keyboard navigation support

### Browser Compatibility
- Modern browsers with ES6 support
- Canvas API support for Word Cloud Mode
- Local storage for data persistence
- File download API for export features

## üìö Getting Started

1. **Access Writing Modes**: Click the "Modes" button in the editor toolbar
2. **Browse Available Modes**: Review mode cards with descriptions
3. **Activate a Mode**: Click "Activate" on any mode card
4. **Use Mode Tools**: Interact with mode-specific controls in the toolbar
5. **Deactivate Mode**: Click the "X" button in the active mode indicator

### Tips for Success
- **Start simple**: Begin with Reader or Meta modes to familiarize yourself
- **Combine modes**: Use multiple modes in sequence for comprehensive analysis
- **Export data**: Save analysis results for future reference
- **Experiment freely**: Most modes support undo/redo or non-destructive editing
